Exercise 03: Custom nginx Image

このページは、自分で build した nginx image から配信されています
演習 02 では bind mount で既成イメージを使いましたが、ここでは Dockerfile を書いて独自イメージを build しました。

演習の流れ

  1. Dockerfile を書く（既成の nginx:alpine をベースに、index.html を追加）
  2. docker build でイメージを build
  3. build したイメージで docker run を実行

使ったコマンド

docker build -t my-custom-nginx:latest .
docker run --rm -p 8080:80 my-custom-nginx:latest

Dockerfile の内容

FROM nginx:alpine
COPY index.html /usr/share/nginx/html/

このやり方の特徴

  • ファイル固定化: イメージのビルド時に、index.html が image に組み込まれる
  • 再現性: 別マシンで同じイメージを pull すれば、同じコンテンツが配信される
  • 配布可能: docker push で image をレジストリにシェアできる
  • 本番向き: 開発環境と本番環境で同じ image を使える

演習 02 との違い

項目 演習 02 (bind mount) 演習 03 (build)
image 既成の nginx:alpine をそのまま 独自に build した my-custom-nginx
ファイル ホスト側のファイルを参照（動的） image に組み込まれている（静的）
ファイル変更時 即座に反映 build し直す必要がある
用途 開発環境 本番環境・チーム共有

次のステップ

  • Dockerfile に nginx.conf をカスタマイズして COPY する
  • 複数ファイル（CSS、JS など）を COPY する
  • 健康チェック（HEALTHCHECK）を追加する
  • image を Docker Hub にシェアする
