# Docker Exercise 01

## 目的

最小のカスタムイメージを作り、Dockerfile が何を定義しているのかを理解する。

## どういう作りになっているか

- `Dockerfile` はイメージの作り方を定義する
- `FROM alpine:3.22` はベースイメージを決める
- `CMD [...]` は、そのイメージから起動したコンテナのデフォルト実行コマンドを決める

この演習は、各命令の役割を見やすくするために意図的に最小構成にしている。

## Build

```bash
docker build -t study-hello .
```

## Run

```bash
docker run --rm study-hello
```

## この設定の意味

- `FROM alpine:3.22`
  - Alpine Linux `3.22` を土台にする
- `CMD ["echo", "hello from docker-k8s-study"]`
  - 明示的に上書きしない限り、このコマンドが起動時に実行される
- `-t study-hello`
  - イメージに分かりやすいタグを付ける
- `--rm`
  - コンテナ終了後に自動削除する

## サポート観点

この演習で答えられるようにするべき問い:

- 問題はビルド段階か、実行段階か
- コンテナは実際に何を実行しているか
- `docker run` で `CMD` を上書きすると何が変わるか

## Q&A

### Q. `alpine` って何？

A. `alpine` は Alpine Linux のこと。軽量な Linux ディストリビューションで、Docker イメージのベースとしてよく使われる。

この演習で `alpine:3.22` を使っている理由:

- イメージサイズが比較的小さい
- 起動確認のような最小演習に向いている
- `FROM` が「どの OS / どの土台から始めるか」を理解しやすい

サポート観点では、`alpine` を使うと次の点も意識する。

- 一般的な Debian/Ubuntu 系とは入っているツールが違う
- パッケージ管理は `apk` を使う
- シェルやライブラリの違いで、手元の想定どおりに動かないことがある

### Q. `alpine` って日本語ではどう発音すればよい？

A. 日本語では普通に `アルパイン` と読めば通じる。英語の発音にかなり厳密に寄せる必要はない。

補足:

- 英語寄りには `アルパイン`
- 日本語の会話でも `アルパイン Linux` で十分通じる
- 少なくとも Docker 文脈で `アルピネ` のようには読まない
