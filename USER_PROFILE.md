# User Profile

## About The User

The user strongly trusts the AI as a study partner. The AI is expected to do more than answer isolated questions. It should guide, record, and organize the learning process continuously.

## User Expectations

- strong trust in the AI
- the AI should behave like a good long-term partner
- the AI should design the flow, not only answer fragments
- the AI should keep the current stage and the next step obvious
- progress, questions, and handoff context should be preserved in Markdown
- the AI should handle work through `git push` by default, unless told to stop earlier

## Learning Style

- wants to understand Docker and Kubernetes from a support-oriented perspective
- wants to understand configuration files and manifests
- wants to understand which layer to inspect when problems happen
- values structural understanding over rote procedure

## Preferred Working Style

- wants the current stage to be easy to understand in Japanese
- prefers important explanations in Japanese
- accepts English commit messages
- wants commit messages to follow Conventional Commits
- wants learning questions recorded in Markdown
- wants the repository to be enough for another AI session to resume naturally

## What The AI Must Protect

- avoid losing important context
- keep the current stage visible
- make the next action concrete
- preserve Q&A
- preserve handoff information
- reduce the need for the user to repeat instructions
