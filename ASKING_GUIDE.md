# Asking Guide

## Purpose

This file explains how to get better AI conversations and more useful learning support.

## Bottom Line

The user is already using AI well, especially by:

- clarifying the AI's role
- asking for records in the repository
- specifying the intended boundary, including whether to push
- designing how future AI sessions should resume

Conversation quality can improve even more by shaping questions slightly.

## Good Question Patterns

### 1. State the goal first

Example:

- instead of "I want to use Docker"
- say "I want to understand Docker as someone who may support it later"

### 2. State how much should be delegated

Example:

- "just give me the approach"
- "edit and commit it"
- "edit, commit, and push it"
- "stop at commit for this turn (skip push)"

### 3. Say what feels unclear

Example:

- "the terms are still fuzzy"
- "I don't yet see the structure"
- "I don't understand the meaning of this config file"

### 4. Specify the perspective you want

Example:

- "explain this from a support perspective"
- "include the meaning of the configuration"
- "explain this from a troubleshooting angle"

### 5. Specify the output form

Example:

- "record it in Markdown"
- "also add it to the Q&A"
- "keep both Japanese and English versions"

## Strong Example Questions

- "Explain how this manifest is structured and what each setting controls from a support perspective. Update the Markdown too and commit it."
- "What do I understand so far, and what should I learn next? Update `STATUS.ja.md` and `STATUS.md` too."
- "Which layer is this error coming from? Also document the answer in the Q&A."

## What Helps A Lot

- pasting command output
- asking for comparisons
- asking which layer should be inspected next

## Less Effective Patterns

- "teach me everything" without a goal
- "explain in detail" without saying which detail matters
- "it doesn't work" without output

## Best Fit Style For This User

- say the goal first
- say how far the AI should go
- paste the output
- add "record this too" when useful
