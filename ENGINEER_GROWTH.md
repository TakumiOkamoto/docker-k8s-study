# Engineer Growth Guide

## Purpose

This file captures what is worth paying attention to when growing into a modern engineer, beyond only learning Docker and Kubernetes.

## Bottom Line

A strong engineer today is not only someone who can write code. A stronger engineer can also:

- understand structure
- isolate problems
- preserve knowledge
- use AI effectively
- turn learning into an ongoing system

## Important Areas To Develop

### 1. Structural understanding over procedure memorization

Do not stop at remembering commands. Be able to explain:

- what runs where
- which configuration controls what
- which layer can fail

### 2. Troubleshooting ability

Modern work rewards the ability to inspect an unfamiliar problem in a clear order.

Useful habits:

- observe symptoms carefully
- separate layers
- confirm reproduction conditions
- connect logs to configuration

### 3. Documentation habit

In both learning and work, value increases when something can be revisited and reproduced later.

Preserve:

- what was done
- why it was done
- what became clear
- what is still unclear

### 4. Ability to work in both English and Japanese

Primary technical sources are often in English, while personal understanding may be faster in Japanese. Being able to move between both is a real advantage.

### 5. Skill in using AI well

Do not only ask AI for answers. Use it to:

- clarify goals
- inspect outputs
- preserve notes
- organize next steps

### 6. Small but continuous learning

Long-term growth is usually stronger through:

- small exercises
- small commits
- small Q&A entries

than through oversized bursts of effort.

### 7. Move from user perspective to support perspective

Higher engineering value often comes from seeing not only:

- how to use something

but also:

- why it behaves that way
- where it tends to break
- how to restore it

## How This Connects To This Repository

This repository turns those ideas into practice:

- `STATUS.*` keeps the current stage visible
- `QA.*` accumulates understanding
- `HANDOFF.*` supports future sessions
- `notes/` and `exercises/` explain structure and configuration meaning

## Note For Future AI Sessions

This guide is not generic decoration. It reflects the intended growth direction for this user. Future guidance should keep pushing toward structural understanding, troubleshooting, documentation, and effective AI collaboration.
